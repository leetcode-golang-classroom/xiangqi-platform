## 觀察與省錢建議

1. **最貴的不是難的功能，而是長的功能。** 「棋子 Kind 對應」($126) 與「載入棋譜＋選單分離」($89) 之所以最貴，是因為當時對話最長、迭代最多回合，每回合都付高額 cache read——不代表這些功能最複雜。
2. **晚期的小功能極便宜。** 結束遊戲／難度調整／名稱顯示各僅 $2–6，因為已在 `/compact` 後、上下文短。
3. **善用 `/compact` 與拆分 session。** 今天若在幾個里程碑後就壓縮或開新 session，cache read 成本可明顯下降（這是費用的最大宗，$428）。
4. **產出 token 佔比小但單價最高（$75/M）。** 全日產出 187 萬 ≈ $140；控制成本主要靠縮短上下文，而非少寫程式碼。
